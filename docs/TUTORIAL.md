## 1. 系统部署

### 1.1 Docker 部署（推荐）

```bash
git clone https://github.com/MemoryKD/fsae-plm.git
cd fsae-plm
docker-compose up -d
```

启动后访问：
- 前端页面：http://localhost
- API 文档：http://localhost/api/docs

### 1.2 本地开发模式

**后端：**

```bash
cd backend
pip install -r requirements.txt
echo "DATABASE_URL=sqlite+aiosqlite:///./fsae_plm.db" > .env
echo "SECRET_KEY=your-secret-key" >> .env
uvicorn app.main:app --reload --port 8000
```

**前端：**

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:3000

**CATIA 客户端：**

```bash
cd catia_client
pip install -r requirements.txt
python main.py
```

## 2. 用户注册与登录

### 2.1 注册新账号

1. 打开系统首页，自动跳转到登录页面
2. 点击 **注册** 按钮
3. 填写信息：
   - **用户名**：唯一的登录名（如 `zhangsan`）
   - **密码**：至少 6 位
   - **角色**：选择 `designer`（设计师）或 `viewer`（查看者）
   - **团队**：可选，填写所属组名（如 `悬架组`）
4. 点击 **注册** 完成

### 2.2 登录

在登录页面输入用户名和密码，点击 **登录**。

### 2.3 角色权限说明

| 角色 | 权限 |
|------|------|
| `admin`（管理员） | 所有功能：用户管理、模板管理、零件管理 |
| `manager`（组长） | 管理本组零件、审核更改通告 |
| `designer`（设计师） | 创建/编辑零件、检入/检出、上传版本 |
| `viewer`（查看者） | 只读查看零件和 BOM |

## 3. 零件管理

### 3.1 查看零件列表

登录后进入零件列表页面：

- **搜索**：在搜索框输入零件编号或名称，支持模糊搜索
- **缩略图**：每个零件卡片显示缩略图预览
- **状态标签**：显示生命周期状态（工作中/已发布）和检入检出状态

### 3.2 创建新零件

**方式一：手动创建（网页端）**

1. 点击页面顶部 **新建零件** 按钮
2. 填写零件信息：
   - **零件编号**：如 `FS-SUS-001`
   - **零件名称**：如 `前下摆臂`
   - **类型**：选择 `part`（零件）或 `assembly`（装配体）
   - **子系统**：选择所属子系统
3. 点击 **确定** 创建

**方式二：智能创建（CATIA 客户端）**

1. 在 CATIA 中打开零件文件
2. 打开 CATIA 客户端，点击 **新建零件**
3. 点击 **从 CATIA** 按钮自动读取当前文件
4. 选择编号模板和子系统，系统自动预览零件号
5. 点击 **创建并上传**

智能创建会自动：
- 根据模板生成下一个可用零件号
- 重命名文件为零件号
- 上传到服务器
- 写入 CATIA 自定义属性

### 3.3 查看零件详情

点击零件列表中的任意零件，进入详情页面：

- 基本信息（编号、名称、类型、子系统）
- 缩略图预览
- 当前版本号
- 生命周期状态和检入检出状态
- 操作按钮（检入/检出/发布/取消发布）
- 版本历史列表

## 4. 版本管理（检入/检出）

### 4.1 版本格式

版本号格式为 `{大版本}.{小版本}`，如 `A.1`、`A.2`、`B.1`。

### 4.2 检出

1. 进入零件详情页面（零件状态为"检入"时可操作）
2. 点击 **检出** 按钮
3. 系统自动从服务器下载最新版本文件
4. 选择本地保存位置
5. 可选：在 CATIA 中打开文件

检出后：
- 零件状态变为"检出"
- 其他用户可以看到该零件已被检出（仅提示，不锁定）

### 4.3 检入

1. 在 CATIA 中完成编辑并保存
2. 进入零件详情页面（零件状态为"检出"时可操作）
3. 点击 **检入** 按钮
4. 选择编辑后的文件
5. 填写检入备注（可选，如"修改了安装孔位置"）
6. 点击确定
7. 文件上传到服务器
8. 系统提示是否删除本地文件

检入后：
- 小版本自动递增：`A.1` → `A.2`
- 零件状态恢复为"检入"
- 版本历史记录新增一条

### 4.4 版本递增规则

| 操作 | 版本变化 | 说明 |
|------|----------|------|
| 创建零件 | → `A.1` | 初始版本 |
| 检入 | `A.1` → `A.2` → `A.3`... | 每次检入递增小版本 |
| 取消发布 | `A.2` → `B.1` | 递增大版本，小版本重置 |

## 5. 生命周期管理

### 5.1 两种生命周期状态

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| **工作中** | 零件正在设计/修改 | 检入、检出、发布 |
| **已发布** | 零件已定型 | 取消发布（需更改通告） |

### 5.2 发布

1. 零件状态必须为"检入"（不能在检出状态下发布）
2. 进入零件详情页面
3. 点击 **发布** 按钮
4. 零件状态变为"已发布"

### 5.3 取消发布

已发布零件如需修改，必须先创建更改通告：

1. 进入零件详情页面
2. 点击 **取消发布** 按钮
3. 在弹出的更改通告对话框中填写：
   - **通告标题**：如"修改前摆臂安装孔"
   - **变更原因**：如"装配干涉"
   - **详细说明**：可选
4. 点击 **确认取消发布**
5. 系统自动创建更改通告 → 批准 → 执行取消发布
6. 大版本递增：`A.2` → `B.1`

## 6. 更改通告

### 6.1 什么是更改通告

更改通告（Engineering Change Notice, ECN）用于管理已发布零件的变更。任何已发布零件要回到工作状态，都必须经过更改通告流程。

### 6.2 创建更改通告

在取消发布时自动创建，或通过 API 手动创建：

- **通告编号**：自动生成，格式 `ECN-{年份}-{序号}`，如 `ECN-2026-0001`
- **标题**：变更的简要描述
- **原因**：为什么要变更
- **状态**：待审批 → 已批准 → 已完成

### 6.3 审批流程

```
创建通告（待审批）→ 审批（已批准/已拒绝）→ 取消发布（已完成）
```

## 7. 编号规则模板

> 需要管理员权限

### 7.1 查看模板列表

1. 侧边栏点击 **编号模板**
2. 查看所有已创建的编号模板

### 7.2 创建编号模板

1. 点击 **新建模板** 按钮
2. 填写模板配置：
   - **模板名称**：如 `标准零件编号`
   - **前缀**：如 `FS`（FSAE 缩写）
   - **分隔符**：如 `-`
   - **序号位数**：如 `3`（生成 001、002、003...）
   - **子系统代码**：JSON 格式，如：
     ```json
     {
       "SUS": "悬架",
       "BDY": "车身",
       "PWT": "动力总成",
       "ELC": "电气",
       "AER": "空气动力学"
     }
     ```
3. 点击 **保存**

### 7.3 编号生成规则

模板配置示例：前缀 `FS`，分隔符 `-`，序号位数 `3`

生成结果：`FS-SUS-001`、`FS-SUS-002`、`FS-BDY-001` ...

格式：`{前缀}-{子系统代码}-{序号}`

## 9. CATIA 客户端使用

### 9.1 安装

```bash
cd catia_client
pip install -r requirements.txt
```

或双击运行 `install.bat`。

依赖：ttkbootstrap、requests、Pillow、pywin32

### 9.2 启动

1. 先启动 CATIA V5
2. 运行 `python catia_client/main.py`
3. 程序会自动尝试连接 CATIA

### 9.3 登录

1. 输入服务器地址（如 `http://localhost:8000`）
2. 输入用户名和密码
3. 勾选"记住我"可保存登录信息
4. 点击 **登录**

### 9.4 浏览零件

登录后主界面显示所有零件：

- **搜索**：顶部搜索框支持按零件号、名称搜索
- **缩略图卡片**：每个零件显示缩略图、零件号、名称、状态
- **点击卡片**：打开零件详情对话框

### 9.5 创建零件

1. 点击 **新建零件** 按钮
2. 选择文件来源：
   - **选择文件**：从本地文件系统选择
   - **从 CATIA**：自动读取当前 CATIA 打开的文件
3. 填写零件名称
4. 选择类型（part/assembly/document）
5. 选择编号模板
6. 选择子系统
7. 预览自动生成的零件号
8. 点击 **创建并上传**

### 9.6 检入/检出

**检出：**
1. 在零件详情对话框中点击 **检出**
2. 选择本地保存路径
3. 文件从服务器下载到本地
4. 可选在 CATIA 中打开

**检入：**
1. 在 CATIA 中编辑并保存
2. 在零件详情对话框中点击 **检入**
3. 选择编辑后的文件
4. 填写备注
5. 文件上传到服务器
6. 提示是否删除本地文件

### 9.7 发布/取消发布

**发布：**
- 零件状态为"检入"时，点击 **发布**

**取消发布：**
1. 点击 **取消发布**
2. 填写更改通告信息（标题、原因）
3. 系统自动创建并批准通告
4. 执行取消发布，大版本递增

### 9.8 CATIA 属性同步

客户端会自动在 CATIA 文件中写入以下自定义属性：

| 属性名 | 说明 |
|--------|------|
| `PLM_PartNumber` | 零件编号 |
| `PLM_Version` | 当前版本（如 A.1） |
| `PLM_State` | 生命周期状态 |
| `PLM_Name` | 零件名称 |

每次版本变更都会自动更新这些属性。

## 11. API 文档

系统自动生成交互式 API 文档：

### Swagger UI

访问 http://localhost:8000/docs

### 主要 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 登录 |
| GET | `/api/parts/` | 零件列表（支持 `?search=`） |
| POST | `/api/parts/auto-create` | 自动创建零件 |
| POST | `/api/parts/{id}/checkout` | 检出 |
| POST | `/api/parts/{id}/checkin` | 检入（multipart） |
| POST | `/api/parts/{id}/publish` | 发布 |
| POST | `/api/parts/{id}/unpublish` | 取消发布 |
| GET | `/api/parts/{id}/download` | 下载文件 |
| GET | `/api/parts/{id}/thumbnail` | 获取缩略图 |
| POST | `/api/change-notices/` | 创建更改通告 |
| GET | `/api/templates/` | 编号模板列表 |
| GET | `/api/users/` | 用户列表（管理员） |

### 认证方式

大多数 API 需要在请求头中携带 JWT token：

```
Authorization: Bearer <your-token>
```

获取 token：调用 `POST /api/auth/login`，响应中的 `access_token` 即为 token。

## 常见问题

**Q: 忘记密码怎么办？**
A: 目前不支持自助重置密码，请联系管理员在数据库中重置。

**Q: 上传文件大小有限制吗？**
A: 默认最大 100MB（Nginx 配置）。本地开发无限制。

**Q: 支持哪些文件格式？**
A: 支持任意文件格式。常用：`.CATPart`、`.CATProduct`、`.CATDrawing`、`.pdf`、`.docx`。

**Q: 检出后别人能修改吗？**
A: 系统采用"仅提示"策略，不锁定文件。其他用户可以看到零件已被谁检出，但仍可操作。

**Q: 取消发布为什么需要更改通告？**
A: 已发布零件代表定型状态，变更需要记录原因，确保变更可追溯。

**Q: CATIA 客户端连不上 CATIA？**
A: 确保 CATIA V5 已启动，且 pywin32 已安装。客户端启动时会自动尝试连接。

**Q: 如何备份数据？**
A: Docker 部署时：
```bash
docker exec postgres pg_dump -U postgres fsae_plm > backup.sql
```
