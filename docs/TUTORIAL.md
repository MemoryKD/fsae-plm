## 1. 系统部署

### 1.1 Docker 部署（推荐）

```bash
git clone https://github.com/MemoryKD/fsae-plm.git
cd fsae-plm
docker-compose up -d
```

启动后访问：
- 前端页面：http://localhost
- API 文档：http://localhost/api/docs

Docker 会启动 5 个容器：
- **nginx**：反向代理，端口 80
- **backend**：FastAPI 后端
- **frontend**：Vue 3 前端（构建后由 nginx 提供静态文件）
- **postgres**：PostgreSQL 数据库
- **redis**：缓存

### 1.2 本地开发模式

**后端：**

```bash
cd backend
pip install -r requirements.txt
echo "DATABASE_URL=sqlite+aiosqlite:///./fsae_plm.db" > .env
echo "SECRET_KEY=your-secret-key" >> .env
uvicorn app.main:app --reload --port 8000
```

**前端：**

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:3000

**C# CATIA 客户端：**

```bash
cd catia_client_cs/CatiaClient
dotnet build
```

或在 Visual Studio 中打开 `CatiaClient.sln`。需要 .NET 8.0 SDK。

## 2. 用户注册与登录

### 2.1 注册新账号

1. 打开系统首页，自动跳转到登录页面
2. 点击 **注册** 按钮（或「去注册」链接）
3. 填写信息：
   - **用户名**：唯一的登录名（如 `zhangsan`）
   - **密码**：至少 6 位
   - **姓名**：真实姓名（必填）
   - **部门**：所属部门（如 `悬架组`）
   - **加入年份**：如 `2024`
   - **联系电话**：手机号
4. 点击 **注册** 提交

### 2.2 审批流程

注册后账号状态为"待审批"，**不能立即登录**。

管理员操作：
1. 登录管理员账号
2. 侧边栏点击 **待审批用户**
3. 查看待审批用户列表
4. 点击 **批准** 或 **拒绝**

批准后用户即可正常登录。

### 2.3 登录

在登录页面输入用户名和密码，点击 **登录**。

- 如果账号还在审批中，会提示"账号正在等待审批"
- 如果账号被拒绝，会提示"账号已被拒绝"

### 2.4 默认账号

系统初始有一个管理员账号：
- 用户名：`admin`
- 密码：`admin123`

### 2.5 角色权限说明

| 角色 | 权限 |
|------|------|
| `admin`（管理员） | 所有功能：用户管理、模板管理、角色管理 |
| `manager`（组长） | 管理零件、审核更改通告 |
| `designer`（设计师） | 创建/编辑零件、检入/检出、上传版本 |
| `viewer`（查看者） | 只读查看零件和 BOM |

## 3. 零件管理

### 3.1 查看零件列表

登录后进入零件列表页面：

- **搜索**：在搜索框输入零件编号或名称，支持模糊搜索
- **缩略图**：每个零件卡片显示缩略图预览
- **状态标签**：显示生命周期状态（工作中/已发布）和检入检出状态
- **分支标签**：显示分支名称或"原生"标记

### 3.2 创建新零件

**方式一：手动创建（网页端）**

1. 点击页面顶部 **新建零件** 按钮
2. 填写零件信息：
   - **零件编号**：如 `FS-SUS-001`
   - **零件名称**：如 `前下摆臂`
   - **类型**：选择 `part`（零件）或 `assembly`（装配体）
   - **子系统**：选择所属子系统
3. 点击 **确定** 创建

**方式二：模板创建（网页端）**

1. 点击 **使用模板创建**
2. 选择编号模板
3. 选择子系统和类型
4. 系统自动预览零件号（如 `FS-SUS-001`）
5. 点击 **创建**

**方式三：智能创建（CATIA 客户端）**

1. 在 CATIA 中打开零件文件
2. 打开 CATIA 客户端，点击 **新建零件**
3. 点击 **从 CATIA** 按钮自动读取当前文件
4. 选择编号模板和子系统，系统自动预览零件号
5. 点击 **创建并上传**

智能创建会自动：
- 根据模板生成下一个可用零件号
- 上传到服务器
- 写入 CATIA 自定义属性

### 3.3 查看零件详情

点击零件列表中的任意零件，进入详情页面：

- 基本信息（编号、名称、类型、子系统）
- 缩略图预览
- 3D 模型预览（支持 CATIA/STEP 文件转 glTF）
- 当前版本号
- 生命周期状态和检入检出状态
- 派生来源和分支信息
- 操作按钮（检入/检出/发布/取消发布/创建分支）
- 版本历史列表
- BOM 列表

### 3.4 删除零件

> 需要 `delete_parts` 权限

在零件列表中点击删除按钮，确认后删除。已检出的零件不能删除。

## 4. 版本管理（检入/检出）

### 4.1 版本格式

版本号格式为 `{大版本}.{小版本}`，如 `A.1`、`A.2`、`B.1`。

### 4.2 检出

1. 进入零件详情页面（零件状态为"检入"时可操作）
2. 点击 **检出** 按钮
3. 系统自动从服务器下载最新版本文件
4. 选择本地保存位置
5. 可选：在 CATIA 中打开文件

检出后：
- 零件状态变为"检出"
- 其他用户可以看到该零件已被谁检出

### 4.3 检入

1. 在 CATIA 中完成编辑并保存
2. 进入零件详情页面（零件状态为"检出"时可操作）
3. 点击 **检入** 按钮
4. 选择编辑后的文件
5. 填写检入备注（可选，如"修改了安装孔位置"）
6. 点击确定
7. 文件上传到服务器

检入后：
- 小版本自动递增：`A.1` → `A.2`
- 零件状态恢复为"检入"
- 版本历史记录新增一条

### 4.4 版本递增规则

| 操作 | 版本变化 | 说明 |
|------|----------|------|
| 创建零件 | → `A.1` | 初始版本 |
| 检入 | `A.1` → `A.2` → `A.3`... | 每次检入递增小版本 |
| 取消发布 | `A.2` → `B.1` | 递增大版本，小版本重置 |

### 4.5 装配体检入/检出规则

- 检出装配体时，所有子零件也会被标记为检出
- 检入装配体时，所有子零件必须先检入
- 下载装配体会打包所有 BOM 中的零件为 ZIP

## 5. 生命周期管理

### 5.1 两种生命周期状态

| 状态 | 说明 | 可执行操作 |
|------|------|-----------|
| **工作中** | 零件正在设计/修改 | 检入、检出、发布 |
| **已发布** | 零件已定型 | 取消发布（需更改通告） |

### 5.2 发布

1. 零件状态必须为"检入"（不能在检出状态下发布）
2. 进入零件详情页面
3. 点击 **发布** 按钮
4. 零件状态变为"已发布"

### 5.3 取消发布

已发布零件如需修改，必须先创建更改通告：

1. 进入零件详情页面
2. 点击 **取消发布** 按钮
3. 在弹出的更改通告对话框中填写：
   - **通告标题**：如"修改前摆臂安装孔"
   - **变更原因**：如"装配干涉"
4. 点击 **确认取消发布**
5. 系统自动创建更改通告
6. 大版本递增：`A.2` → `B.1`

## 6. 更改通告

### 6.1 什么是更改通告

更改通告（Engineering Change Notice, ECN）用于管理已发布零件的变更。任何已发布零件要回到工作状态，都必须经过更改通告流程。

### 6.2 创建更改通告

在取消发布时自动创建，或通过 API 手动创建：

- **通告编号**：自动生成，格式 `ECN-{年份}-{序号}`，如 `ECN-2026-0001`
- **标题**：变更的简要描述
- **原因**：为什么要变更
- **状态**：待审批 → 已批准 → 已完成

### 6.3 审批流程

```
创建通告（待审批）→ 审批（已批准/已拒绝）→ 取消发布（已完成）
```

管理员在侧边栏「待审批」页面可以查看和审批更改通告。

## 7. 编号规则模板

> 需要管理员权限

### 7.1 查看模板列表

1. 侧边栏点击 **编号规则**
2. 查看所有已创建的编号模板

### 7.2 创建编号模板

1. 点击 **新建模板** 按钮
2. 填写模板配置：
   - **模板名称**：如 `标准零件编号`
   - **前缀**：如 `FS`（FSAE 缩写）
   - **分隔符**：如 `-`
   - **序号位数**：如 `3`（生成 001、002、003...）
   - **子系统代码**：如 `SUS=悬架, BDY=车身, PWT=动力总成`
   - **类型代码**：如 `P=零件, A=装配体`
3. 点击 **保存**

### 7.3 拖拽排序

子系统代码和类型代码支持拖拽排序，调整显示顺序。

### 7.4 编号生成规则

模板配置示例：前缀 `FS`，分隔符 `-`，序号位数 `3`

生成结果：`FS-SUS-P-001`、`FS-SUS-P-002`、`FS-BDY-A-001` ...

格式：`{前缀}-{子系统代码}-{类型代码}-{序号}`

## 8. BOM 管理

### 8.1 查看 BOM

1. 进入装配体类型的零件详情页面
2. 查看该装配体包含的所有子零件

### 8.2 添加 BOM 项

1. 在 BOM 页面点击 **添加零件**
2. 输入子零件编号或从列表选择

### 8.3 删除 BOM 项

在 BOM 列表中点击删除按钮，确认后删除。

### 8.4 导出 BOM

1. 在 BOM 页面点击 **导出 Excel**
2. 浏览器自动下载 `.xlsx` 文件

### 8.5 整包下载

在零件详情页面点击 **下载全部**，系统会将装配体及所有 BOM 中的零件打包为 ZIP 文件下载。

## 12. C# CATIA 客户端使用

C# WPF 客户端是推荐的 CATIA 集成方案，基于 .NET 8 和 MVVM 架构。

### 12.1 安装

1. 安装 .NET 8.0 SDK
2. 打开 `catia_client_cs/CatiaClient.sln`（Visual Studio）
3. 构建项目（F6）
4. 运行程序（F5）

或命令行：
```bash
cd catia_client_cs/CatiaClient
dotnet build
dotnet run
```

### 12.2 CATIA 连接

客户端底部状态栏实时显示 CATIA 连接状态：
- **绿灯**：CATIA 已连接
- **红灯**：CATIA 未连接

连接失败时会弹窗显示详细错误信息。点击 **重新连接** 按钮可重新尝试连接。

> 注意：CATIA 必须在客户端启动前已运行。客户端通过 COM 自动化连接 CATIA，支持 CATIA V5/V6。

### 12.3 登录

1. 启动客户端
2. 填写服务器地址（默认 `http://localhost/api`）
3. 输入用户名和密码
4. 点击 **登录**

### 12.4 注册

1. 在登录页面点击 **注册账号**
2. 填写用户名、密码、姓名、部门、加入年份、联系电话
3. 点击 **注册**
4. 显示"注册成功，请等待管理员审批"
5. 管理员在网页端批准后即可登录

### 12.5 浏览零件

登录后主界面显示所有零件：
- **搜索**：顶部搜索框支持按零件号、名称搜索
- **刷新**：重新加载零件列表
- **列表**：每个零件显示零件号、名称、类型、版本、生命周期状态、检入检出状态、分支
- **点击**：打开零件详情

### 12.6 创建零件

1. 点击 **新建零件** 按钮
2. 填写零件名称
3. 选择类型（part / assembly / document）
4. 选择编号模板
5. 选择子系统
6. 预览自动生成的零件号
7. 点击 **创建**
8. 创建成功后弹出提示："是否立即选择文件导入到系统？"
9. 选择 **是** → 弹出文件选择器，选择本地 CATIA 文件后自动检入

### 12.7 检出

1. 在零件详情中点击 **检出**（仅"检入"状态可操作）
2. 系统锁定零件，下载最新版本文件
3. CATIA 已连接 → 自动在 CATIA 中打开文件
4. CATIA 未连接 → 弹出"另存为"对话框，用户选择保存位置

### 12.8 检入

1. 在 CATIA 中编辑并保存
2. 在零件详情中点击 **检入**（仅"检出"状态可操作）
3. CATIA 已连接 → 自动读取当前打开的文档
4. CATIA 未连接 → 弹出文件选择器，手动选择文件
5. 文件上传到服务器，小版本自动递增

**文件校验规则**（v1.3.0 新增）：
- 扩展名必须是合法 CAD 格式（.CATPart, .CATProduct, .step, .stp, .stl 等）
- 文件类型必须与上一版本一致（不能把 .txt 检入到 .CATPart 零件中）
- 文件内容不能与上一版本完全相同（SHA-256 哈希比对）

### 12.9 发布

零件状态为"检入"时，点击 **发布**。发布后零件变为只读状态。

### 12.10 CATIA 属性同步

点击 **同步属性到 CATIA** 按钮，将 PLM 属性写入 CATIA 文档的自定义属性：

| 属性名 | 说明 |
|--------|------|
| `PLM_PartNumber` | 零件编号 |
| `PLM_Version` | 当前版本（如 A.1） |
| `PLM_State` | 生命周期状态 |
| `PLM_Name` | 零件名称 |

与 C# 客户端相同：登录、浏览零件、创建零件、检入/检出、发布、属性同步、缩略图截取。

## 15. API 文档

### 认证方式

大多数 API 需要在请求头中携带 JWT token：

## 16. 常见问题

**Q: 注册后无法登录？**
A: 新注册的账号需要管理员审批。请通知管理员在「待审批用户」页面批准你的账号。

**Q: 忘记密码怎么办？**
A: 目前不支持自助重置密码，请联系管理员在数据库中重置。

**Q: 上传文件大小有限制吗？**
A: 默认最大 100MB（Nginx 配置）。本地开发无限制。

**Q: 支持哪些文件格式？**
A: 支持任意文件格式。常用：`.CATPart`、`.CATProduct`、`.CATDrawing`、`.pdf`、`.docx`。

**Q: 检出后别人能修改吗？**
A: 系统采用"仅提示"策略，不锁定文件。其他用户可以看到零件已被谁检出，但仍可操作。

**Q: 取消发布为什么需要更改通告？**
A: 已发布零件代表定型状态，变更需要记录原因，确保变更可追溯。

**Q: CATIA 客户端连不上 CATIA？**
A: 确保 CATIA V5 已启动。C# 客户端使用 P/Invoke 连接 CATIA COM，Python 客户端使用 pywin32。

**Q: 3D 预览不工作？**
A: 3D 预览需要 FreeCAD CLI 进行文件转换。确保 FreeCAD 已安装并在 PATH 中。

**Q: 如何备份数据？**
A: Docker 部署时：
```bash
docker exec fsae-plm-postgres-1 pg_dump -U postgres fsae_plm > backup.sql
```

**Q: C# 客户端需要什么环境？**
A: 需要 Windows 系统、.NET 8.0 SDK、CATIA V5。
