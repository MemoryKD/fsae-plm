## 1. 系统部署

### 1.1 Docker 部署（推荐）

```bash
git clone https://github.com/MemoryKD/fsae-plm.git
cd fsae-plm
docker-compose up -d
```

启动后访问：
- 前端页面：http://localhost
- API 文档：http://localhost/api/docs

### 1.2 本地开发模式

**后端：**

```bash
cd backend
pip install -r requirements.txt
# 创建 .env 文件
echo "DATABASE_URL=sqlite+aiosqlite:///./fsae_plm.db" > .env
echo "SECRET_KEY=your-secret-key" >> .env
# 启动
uvicorn app.main:app --reload --port 8000
```

**前端：**

```bash
cd frontend
npm install
npm run dev
```

访问 http://localhost:3000

## 2. 用户注册与登录

### 2.1 注册新账号

1. 打开系统首页，自动跳转到登录页面
2. 点击 **注册** 按钮
3. 填写信息：
   - **用户名**：唯一的登录名（如 `zhangsan`）
   - **密码**：至少 6 位
   - **角色**：选择 `designer`（设计师）或 `viewer`（查看者）
   - **团队**：可选，填写所属组名（如 `悬架组`）
4. 点击 **注册** 完成

### 2.2 登录

1. 在登录页面输入用户名和密码
2. 点击 **登录**
3. 登录成功后自动跳转到零件列表页面

### 2.3 角色权限说明

| 角色 | 权限 |
|------|------|
| `admin`（管理员） | 所有功能：用户管理、模板管理、零件管理、审批 |
| `manager`（组长） | 管理本组零件、审核零件状态流转 |
| `designer`（设计师） | 创建/编辑自己负责的零件、上传版本 |
| `viewer`（查看者） | 只读查看零件和 BOM |

## 3. 零件管理

### 3.1 查看零件列表

登录后进入零件列表页面，可以看到所有零件：

- **搜索**：在搜索框输入零件编号或名称，点击搜索
- **分页**：底部翻页控件浏览更多零件

### 3.2 创建新零件

1. 点击页面顶部 **新建零件** 按钮
2. 填写零件信息：
   - **零件编号**：如 `FS-SUS-001`（需符合编号规则）
   - **零件名称**：如 `前下摆臂`
   - **类型**：选择 `part`（零件）或 `assembly`（装配体）
   - **子系统**：选择所属子系统（如 `悬架`、`车身`）
3. 点击 **确定** 创建

创建后零件状态自动设为 `设计中`。

### 3.3 查看零件详情

点击零件列表中的任意零件，进入详情页面，可以看到：

- 基本信息（编号、名称、类型、子系统）
- 当前版本号
- 工作流状态
- 版本历史列表

## 4. 版本管理

### 4.1 上传新版本

1. 进入零件详情页面
2. 点击 **上传版本** 按钮
3. 选择文件（支持 `.CATPart`、`.CATProduct`、`.pdf`、`.docx` 等）
4. 填写版本说明（可选，如 "修改了安装孔位置"）
5. 点击 **上传**

上传成功后：
- 版本号自动递增（A → B → C ...）
- 文件存储到服务器
- 版本历史列表更新

### 4.2 查看版本历史

在零件详情页面下方可以看到所有历史版本：

| 列 | 说明 |
|----|------|
| 版本号 | 自动递增的版本标识 |
| 文件名 | 上传的文件名 |
| 文件大小 | 文件体积 |
| 上传时间 | 上传时间戳 |
| 说明 | 上传时填写的备注 |

### 4.3 下载历史版本

点击版本列表中任意版本的 **下载** 按钮，即可下载对应版本的文件。

## 5. 编号规则模板

> 需要管理员权限

### 5.1 查看模板列表

1. 侧边栏点击 **编号模板**
2. 查看所有已创建的编号模板

### 5.2 创建编号模板

1. 点击 **新建模板** 按钮
2. 填写模板配置：
   - **模板名称**：如 `标准零件编号`
   - **前缀**：如 `FS`（FSAE 缩写）
   - **分隔符**：如 `-`
   - **序号位数**：如 `3`（生成 001、002、003...）
   - **子系统代码**：JSON 格式，如：
     ```json
     {
       "SUS": "悬架",
       "BDY": "车身",
       "PWT": "动力总成",
       "ELC": "电气",
       "AER": "空气动力学"
     }
     ```
3. 点击 **保存**

### 5.3 编号生成规则

模板配置示例：前缀 `FS`，分隔符 `-`，序号位数 `3`

生成结果：`FS-SUS-001`、`FS-SUS-002`、`FS-BDY-001` ...

格式：`{前缀}-{子系统代码}-{序号}`

## 6. 工作流与审批

### 6.1 默认工作流

系统内置默认工作流：

```
设计中 → 审核中 → 已发布
```

### 6.2 状态流转

1. 进入零件详情页面
2. 点击 **状态流转** 按钮
3. 选择目标状态（如 `审核中`）
4. 填写审批意见（可选）
5. 点击 **确认**

状态流转规则：
- `designer` 可以将零件从 `设计中` 提交到 `审核中`
- `manager` 可以将零件从 `审核中` 批准为 `已发布`
- `manager` 可以将零件从 `审核中` 驳回回 `设计中`

### 6.3 查看审批历史

在零件详情页面点击 **审批历史**，可以看到：

| 列 | 说明 |
|----|------|
| 原状态 | 流转前的状态 |
| 目标状态 | 流转后的状态 |
| 操作人 | 执行流转的用户 |
| 意见 | 审批备注 |
| 时间 | 操作时间 |

## 7. BOM 管理

### 7.1 查看 BOM

1. 进入装配体类型的零件详情页面
2. 点击 **BOM** 标签页
3. 查看该装配体包含的所有子零件

BOM 列表显示：
- 子零件编号
- 子零件名称
- 数量
- 层级

### 7.2 添加 BOM 项

1. 在 BOM 页面点击 **添加零件**
2. 输入子零件编号或从列表选择
3. 填写数量（默认 1）
4. 点击 **确认**

### 7.3 导出 BOM

1. 在 BOM 页面点击 **导出 Excel**
2. 浏览器自动下载 `.xlsx` 文件
3. 文件格式参照 GB/T 26099 标准

导出内容包含：
- 装配体信息
- 子零件清单（编号、名称、数量、层级）
- 导出时间

## 9. CATIA 插件使用

### 9.1 安装插件

1. 打开 CATIA V5
2. 菜单 → **工具** → **宏** → **Visual Basic 编辑器**（或 `Alt+F11`）
3. 在 VBA 编辑器中：**插入** → **模块**
4. 将 `catia_plugin/FSAE_PLM_Macro.catvba` 的全部内容复制粘贴到新模块中
5. 关闭 VBA 编辑器

### 9.2 配置服务器地址

在宏代码中修改第一行：

```vbscript
Const PLM_SERVER = "http://your-server:8000"
```

将 `your-server` 替换为实际服务器地址（如 `192.168.1.100` 或 `localhost`）。

### 9.3 保存零件到 PLM

1. 在 CATIA 中打开一个零件（`.CATPart`）或装配体（`.CATProduct`）
2. 菜单 → **工具** → **宏** → **宏...**（`Alt+F8`）
3. 选择宏，点击 **运行**
4. 在弹出对话框中输入 `1`，点击确定
5. 确认弹出的零件信息对话框
6. 等待上传完成提示

上传成功后，零件会自动出现在 PLM 系统中。

### 9.4 检查命名规范

1. 在 CATIA 中打开零件
2. 运行宏，输入 `2`
3. 系统会检查零件编号是否符合 PLM 中配置的编号规则
4. 显示检查结果（通过/不通过）

### 9.5 添加到工具栏（可选）

1. 菜单 → **工具** → **自定义**
2. 选择 **命令** 选项卡
3. 在左侧选择 **宏**
4. 将 `SaveToPLM` 拖动到 CATIA 工具栏
5. 关闭自定义对话框

现在可以直接点击工具栏按钮运行宏。

## 10. API 文档

系统自动生成交互式 API 文档：

### Swagger UI

访问 http://localhost:8000/docs

可以在线测试所有 API 端点：

1. 点击 **Authorize** 按钮，输入 JWT token
2. 展开任意 API 端点
3. 点击 **Try it out**
4. 填写参数，点击 **Execute**

### 主要 API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/register` | 注册 |
| POST | `/api/auth/login` | 登录，返回 JWT token |
| GET | `/api/parts/` | 零件列表（支持 `?search=`） |
| POST | `/api/parts/` | 创建零件 |
| GET | `/api/parts/{id}` | 零件详情 |
| GET | `/api/parts/{id}/versions/` | 版本列表 |
| POST | `/api/parts/{id}/versions/` | 上传新版本 |
| GET | `/api/parts/{id}/bom/` | 获取 BOM |
| POST | `/api/parts/{id}/bom/` | 添加 BOM 项 |
| GET | `/api/parts/{id}/bom/export` | 导出 BOM Excel |
| POST | `/api/parts/{id}/workflow/transition` | 状态流转 |
| GET | `/api/parts/{id}/workflow/history` | 审批历史 |
| GET | `/api/templates/` | 编号模板列表 |
| POST | `/api/templates/` | 创建编号模板 |
| GET | `/api/users/` | 用户列表（管理员） |
| PUT | `/api/users/{id}/role` | 修改用户角色（管理员） |

### 认证方式

大多数 API 需要在请求头中携带 JWT token：

```
Authorization: Bearer <your-token>
```

获取 token：调用 `POST /api/auth/login`，响应中的 `access_token` 即为 token。

## 常见问题

**Q: 忘记密码怎么办？**
A: 目前不支持自助重置密码，请联系管理员在数据库中重置。

**Q: 上传文件大小有限制吗？**
A: 默认最大 100MB（Nginx 配置）。如需调整，修改 `nginx/nginx.conf` 中的 `client_max_body_size`。

**Q: 支持哪些文件格式？**
A: 支持任意文件格式。常用格式：`.CATPart`、`.CATProduct`、`.CATDrawing`、`.pdf`、`.docx`、`.xlsx`。

**Q: 如何备份数据？**
A: Docker 部署时，数据存储在 Docker volume 中。使用以下命令备份：
```bash
docker exec postgres pg_dump -U postgres fsae_plm > backup.sql
```

**Q: CATIA 插件无法连接服务器？**
A: 检查：
1. 服务器地址是否正确（宏中 `PLM_SERVER` 常量）
2. 服务器是否正在运行
3. 防火墙是否放行 8000 端口
4. CATIA 所在电脑能否访问服务器地址
